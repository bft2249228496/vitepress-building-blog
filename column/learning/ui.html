vxe-table 中,遍历表格数组，对每一行的对象数据调用组件封装的方法时尽量使用 for 循环而不是 forEach，因为 forEach 遍历出来的 item 是一个全新的对象，会导致调用方法失败。如果一定要使用 forEach，应该使用 index 来获取每一项而不是用 item。代码示例: ​

js
// 正确代码
nextTick(() => {
  for (let i = 0; i < state.selectedData.length; i++) {
    console.log(refTable.value, "refTable.value++++");
    refTable.value.setCheckboxRow(state.selectedData[i], true);
  }
});
// 正确代码
nextTick(() => {
  if (state.selectedData.length) {
    state.selectedData.forEach((item, index) => {
      refTable.value.setCheckboxRow(state.selectedData[index], true);
    });
  }
});
// 错误代码
nextTick(() => {
  if (state.selectedData.length) {
    state.selectedData.forEach((item, index) => {
      refTable.value.setCheckboxRow(item, true);
    });
  }
});

elementPlus 的 el-table 组件中 show-overflow-tooltip 属性 ​

该属性在以下几种情况下会失效: 1.插槽中手动设置了 css 样式的溢出显示省略号 2.插槽根元素用了 div 标签 3.插槽根元素设置了 display: flex 解决方法:手动设置 css 样式的溢出显示省略号，再手动添加 tooltip 组件

js
<el-table>
	  <el-table-column prop="productName" label="产品名称">
			<template #default="{ row }">
				<el-tooltip placement="top" :visible="row.visible">
					<template #content>
						{{ row.productName }}
					</template>
					<span
						class="productName"
						@mouseenter="row.visible = true"
						@mouseleave="row.visible = false"
					>
						<span class="span-txt">{{ row.productName }}</span>
						<el-icon v-if="row.pictureUrls" color="#1890FF" size="18px">
							&nbsp;
							<PictureFilled />
						</el-icon>
					</span>
				</el-tooltip>
			</template>
		</el-table-column>
	</el-table>
.productName {
	display: flex;
	align-items: center;
	.span-txt {
		overflow: hidden;
		text-overflow: ellipsis;
		white-space: nowrap;
	}
}

表格里的 el-select 的数据 option 如果是每次选择后动态过滤已选择的数据导致数据回显异常 ​

因为过滤已选中数据的原因，回显的时候 value 对应的值实际上已经从 option 里面过滤掉了,无法根据 value 回显对应的 label，就会直接显示成 value 处理思路:直接存两份数据，一份保存过滤前的原始数据 originOption,一份保存过滤后的数据 filterOption;每次下拉框隐藏的时候将 option 赋值为 filterOption，每次回显的时候讲 option 赋值为 originOption

elementUI 中 el-dropdown 组件中的@command 方法优化 ​

因为子组件 el-dropdown-menu 的 command 属性值可以为字符串、数值、对象,可以将 command 属性设为一个对象,将函数作为该对象的方法传入,然后在父组件 el-dropdown 组件中的@command 方法里面直接调用传入对象的方法,即可实现子方法的映射，省去了 if 或者 switch 的这种条件判断

html
<el-dropdown @command="(command) => handleOperationFun(command, row)">
  <el-button link type="primary">操作</el-button>
  <template #dropdown>
    <el-dropdown-menu>
      <el-dropdown-item :command="{ command: handleWithdraw }">
        <el-button link type="default">
          <el-icon><Document /></el-icon>
          撤回
        </el-button>
      </el-dropdown-item>
      <el-dropdown-item :command="{ command: handleCancel }">
        <el-button link type="default">
          <el-icon><Document /></el-icon>
          取消
        </el-button>
      </el-dropdown-item>
      <el-dropdown-item :command="{ command: handleDelete }">
        <el-button link type="default">
          <el-icon><Document /></el-icon>
          删除
        </el-button>
      </el-dropdown-item>
    </el-dropdown-menu>
  </template>
</el-dropdown>
js
// 操作按钮
const handleOperationFun = (command: any, row: any) => {
  console.log(command, row, "操作");
  command.command(row);
};
// 撤回按钮
const handleWithdraw = () => {
  console.log("撤回");
};

// 取消按钮
const handleCancel = () => {
  console.log("取消");
};

// 删除按钮
const handleDelete = (row?: any) => {
  console.log("删除", row);
};

表格中的表单校验 ​

表格中进行表单的自定义校验,并且将表格中当前行的数据作为额外参数传递给自定义校验函数。 关键点：

  1. dom 中结构层级为 el-form>el-table>el-table-column>template>el-form-item
  2. el-table 中 data 绑定的值必须是 el-form 中 model 绑定的对象的属性
  3. el-form-item 中的 prop 属性写成动态变量,三部分拼接而已:model 绑定值的对象的属性名(如 tableDate) + 表格行的序号{$index} + rules 对象中对应的属性名(如 start)
  4. el-form-item 中需要再添加 rules 属性,绑定的值为 rules 的对应属性(如 rules.start)
  5. 表格行 template 里面的具体组件 v-model 的绑定值需要和当前 el-form-item 中 prop 和 rules 绑定的属性名保持一致(如 row.start)
  6. 传递额外参数时：直接在 el-form-item 里面的 rules 绑定的变量 rules.start 的值改成函数返回数组的形式,dom 上写成 rules.start(row)即可传递表格当前行的参数 row 到函数 startValidateFun 里;startValidateFun 函数里面的大致写法为 const startValidateFun = (row: any) => { return [ { required: true, validator: (rule: any, value: any, callback: any) => validatePass(rule, value, callback, row), trigger: 'change' } ] }
html
<el-form :model="formData" :rules="rules" ref="refFromData">
  <el-table :data="formData.tableData" style="width: 100%">
    <el-table-column label="开始时间">
      <template #header>
        <span>
          <span style="color: red; font-size: 16px">*</span>开始时间
        </span>
      </template>
      <template #default="{ row, $index }">
        <el-form-item
          :prop="`tableData.${$index}.start`"
          :rules="rules.start(row)"
        >
          <el-date-picker
            v-model="row.start"
            type="datetime"
            placeholder="请选择时间"
          />
        </el-form-item>
      </template>
    </el-table-column>
    <el-table-column label="结束时间">
      <template #header>

          <span style="color: red; font-size: 16px">*</span>结束时间
        </span>
      </template>

          />

js
setup() {
   const validatePass = (rule: any, value: any, callback: any, row: any) => {
      if (value) {
        const start = new Date(value).getTime()
        const end = new Date(row.end).getTime()
        if (start > end) {
          callback(new Error('开始时间不能大于结束时间'))
        } else {
          callback()
        }
      } else {
        callback(new Error('请选择时间'))
      }
    }
    const validatePass1 = (rule: any, value: any, callback: any, row: any) => {
      if (value) {
        const start = new Date(row.start).getTime()
        const end = new Date(value).getTime()
        if (start > end) {
          callback(new Error('结束时间不能小于开始时间'))
        } else {
          callback()
        }
      } else {
        callback(new Error('请选择时间'))
      }
    }
     const startValidateFun = (row: any) => {
      return [
        {
          required: true,
          validator: (rule: any, value: any, callback: any) => validatePass(rule, value, callback, row),
          trigger: 'change'
        }
      ]
    }
    const endValidateFun = (row: any) => {
      return [
        {
          required: true,
          validator: (rule: any, value: any, callback: any) => validatePass1(rule, value, callback, row),
          trigger: 'change'
        }
      ]
    }
     const state = reactive({
      formData: {
        tableData: []
      },
       rules: {
        start: startValidateFun,
        end: endValidateFun,
        num: [{ required: true, trigger: 'blur', message: '请输入数量' }],
      } as any,
     })
  return {
    ...toRefs(state),
  }
}
