vue3 报错:找不到模块或其相应的类型声明。（Vue 3 can not find module） ​

如果是自定义页面组件路径报错,在 src 目录下新建 global.d.ts 文件

js
declare module '*.vue'{
  import { ComponentOptions} from 'vue'
  const componentOptions: ComponentOptions
  export default componentOptions
}

如果是第三方插件报错，在 src 或者根目录下新建 global.d.ts 文件:写入如下代码,xxx 为模块名。如果有多个模块报类似错误,则写多段以下代码

js
declare module 'xxx' {
  const content: any
  export = content
}

vue3+vite 项目中,实现自动注册路由 ​

1.下载 vite-plugin-pages

js
npm install -D vite-plugin-pages

2.在 vite.config.ts->defineConfig->plugins 里面写入

js
plugins: [
	// 如果写成Pages()，则等价于Pages({pagesDir: 'src/pages'});另外此处src不能写成@
	Pages({
		pagesDir: 'src/views'
	})
],

3.在 router/index.ts 中写入

js
import { createRouter, createWebHistory, RouteRecordRaw } from "vue-router";
import routes from "virtual:generated-pages"; //关键代码
const router = createRouter({
  history: createWebHistory(),
  routes: routes, //关键代码
});
export default router;

vue3 获取 dom 的两种方式 ​

通过 ref 或者 reactive 都可以实现获取 dom

vue
<template>
  <el-form ref="refForm"></el-form>
</template>
// 方式一
<script lang="ts">
import { defineComponent, reactive, toRefs } from "vue";
export default defineComponent({
  name: "Page",
  components: {},
  setup() {
    const state = reactive({
      refForm: null as any,
    });
    console.log(state.refForm);
    return {
      ...toRefs(state),
    };
  },
});
</script>
// 方式二
<script lang="ts">
import { defineComponent, reactive, toRefs } from "vue";
export default defineComponent({
  name: "Page",
  components: {},
  setup() {
    const refForm = ref();
    console.log(refForm.value);
    return {
      refForm,
    };
  },
});
</script>

vue3 中自动引入子组件 ​

插件 unplugin-auto-import 可以解决每次使用组件都要 import 一次的问题。使用步骤如下： 1.下载插件

js
npm i -D unplugin-auto-import

2.修改 vite.config.ts 配置文件

js
import AutoImport from "unplugin-auto-import/vite";
export default defineConfig({
  plugins: [
    ...AutoImport({
      imports: ["vue", "vue-router"], // 自动导入vue和vue-router相关函数
      dts: "src/auto-import.d.ts", // 生成 `auto-import.d.ts` 全局声明
    }),
  ],
});

vue3 中使用 ref 获取子组件或者 dom 失败的原因 ​

1.子组件为弹窗未使用 nextTick() 2.未在父组件里将当前的 ref 变量 return 出去

点击按钮,无法打开弹窗,页面直接卡死 ​

现象:本地环境:点击新增按钮,打开弹窗,显示如下警告:Maximum recursive updates exceeded in component. This means you have a reactive effect that is mutating its own dependencies and thus recursively triggering itself. Possible sources include component template, render function, updated hook or watcher source function.线上环境:点击按钮,无法打开弹窗,页面直接卡死 原因:父组件给子组件传值的下拉框接口数据有问题,存在有 id，没有 name 的数据，导致页面渲染异常. 解决方法:过滤异常数据

前端 http-server 开启本地 https 服务 ​

步骤： 1.全局或者在当前项目中按照 http-server npm i http-server 2.打开当前项目目录在 git bash 中使用以下命令生成一个证书密钥对 key.pem 和 cert.pem，它将有效期约 10 年 openssl req -newkey rsa:2048 -new -nodes -x509 -days 3650 -keyout key.pem -out cert.pem 3.根据提示填写相关信息，重要必填字段为 CN(Common Name):127.0.0.1 4.将项目中生成的 cert.pem 和 key.pem 复制到 dist 文件夹下,命令行打开到 dist 目录下 http-server -S -C cert.pem -o

TS 报错类型“{}”上不存在属性“state” ​

script 标签上未加 setup 属性

js
// 错误写法
<script lang='ts'>
  const state = reactive({
  'aaa': null,
  'bbb': null
})
</script>
// 正确写法
<script setup lang='ts'>
  const state = reactive({
  'aaa': null,
  'bbb': null
})
</script>

前端使用 Live Server 开启本地 https 服务 ​

  1. vscode 中下载 Live Server 插件
  2. 打开插件设置，配置：
    Custom BrowserHostPort
    chromelocalhost8080
  3. 配置 https: ① 安装 mkcert: 打开 windows 的搜索功能搜索 powershell，并以管理员的身份打开;在 powershell 中输入命令:
sh
Get - ExecutionPolicy;

如果系统返回 Restricted，则在执行命令：

sh
Set-ExecutionPolicy AllSigned

选择 A（全是）；执行命令:

sh
Set-ExecutionPolicy Bypass -Scope Process -Force; [System.Net.ServicePointManager]::SecurityProtocol = [System.Net.ServicePointManager]::SecurityProtocol -bor 3072; iex ((New-Object System.Net.WebClient).DownloadString('https://chocolatey.org/install.ps1'))；

执行

sh
choco -?

验证是否安装成功 ； 安装完成后，执行命令

sh
 choco install mkcert

等待安装完成。 ② 安装本地 CA 并将 CA 加入系统的信任清单中: 执行命令

sh
mkcert - install;

本地 CA 在目录 C:\Users\你的用户名\AppData\Local\mkcert 中。 ③ 生成凭证: 进入到要生成凭证的目录，如 D:\https,执行命令

sh
mkcert example.com "\*.example.com" example.test localhost 127.0.0.1 ::1

④ 设置 live server 的 settings-HTTPS:

sh
"liveServer.settings.https": {
"enable": false, //是否启用 https
"cert": "D:\\https\\example.com+5.pem", //之前生成的证书地址
"key": "D:\\https\\example.com+5-key.pem", //之前生成的密钥地址
"passphrase": ""
}
